# 飞书文档批量导出工具 — 技术方案

**项目名称**: feishu-export
**技术栈**: Rust CLI + 飞书开放平台 API
**版本**: v0.1.0

---

## 一、系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      CLI 命令行前端                           │
│   clap (命令解析) + console (彩色输出) + indicatif (进度条)   │
├─────────────────────────────────────────────────────────────┤
│                      Core 核心层                              │
│  ┌──────────────┐ ┌──────────────┐ ┌────────────────────┐  │
│  │ Feishu API   │ │ 导出引擎     │ │ 文件系统           │  │
│  │ Client       │ │ ExportEngine │ │ FileSystem        │  │
│  └──────────────┘ └──────────────┘ └────────────────────┘  │
│  ┌──────────────┐ ┌──────────────┐                          │
│  │ Token Manager│ │ 缓存/断点   │                          │
│  │ (localstore) │ │ CacheStore   │                          │
│  └──────────────┘ └──────────────┘                          │
├─────────────────────────────────────────────────────────────┤
│                      外部依赖                                 │
│   reqwest (HTTP) · tokio (异步) · serde (JSON) · zip (打包)  │
└─────────────────────────────────────────────────────────────┘
```

**设计原则**:
- 无数据库，本地文件系统存储 token 和缓存
- 异步并发导出，边导边下，10分钟窗口内完成
- 单二进制，无外部依赖，即装即用

## 三、API 接口对接

### 3.1 授权相关

**OAuth 2.0 授权码流程**:

```
1. 构造授权 URL
   GET https://accounts.feishu.cn/open-apis/authen/v1/authorize
   ?app_id=<APP_ID>
   &redirect_uri=<REDIRECT_URI>
   &scope=wiki:wiki:readonly wiki:node:read wiki:node:retrieve wiki:space:retrieve drive:drive:readonly drive:export:readonly docx:document:readonly offline_access
   &state=<随机防CSRF>

2. 用户在浏览器授权后，飞书回调 redirect_uri?code=xxx&state=xxx

3. 用 code 换取 token
   POST https://open.feishu.cn/open-apis/authen/v1/oidc/access_token
   Content-Type: application/json
   {
     "grant_type": "authorization_code",
     "code": "<code>"
   }
   Header: Authorization: Bearer <app_access_token>

4. 响应
   {
     "code": 0,
     "data": {
       "access_token": "xxx",
       "refresh_token": "xxx",
       "token_type": "Bearer",
       "expires_in": 23400,        // access_token 有效期（秒）
       "refresh_expires_in": 2592000 // refresh_token 有效期（秒）
     }
   }

5. 定期刷新（过期前用 refresh_token 刷新）
   POST https://open.feishu.cn/open-apis/authen/v1/oidc/refresh_access_token
```

**redirect_uri 设计**: 启动本地 HTTP 服务监听 `http://localhost:8765/callback`，授权成功后飞书回调带 code，本工具自动取 code 完成流程。

### 3.2 知识库浏览

**获取知识空间列表**:

```
GET https://open.feishu.cn/open-apis/wiki/v2/spaces
Authorization: Bearer <user_access_token>

响应 items[]:
  - space_id: 知识空间 ID
  - name: 空间名称
  - description: 描述
  - space_type: "team" | "person" | "my_library"
  - visibility: "public" | "private"
  - open_sharing: "open" | "closed"
```

**获取子节点列表**:

```
GET https://open.feishu.cn/open-apis/wiki/v2/spaces/<space_id>/nodes
Authorization: Bearer <user_access_token>
Query:
  page_size=50
  page_token=<分页token>
  parent_node_token=<父节点token，空=根节点>

响应 items[]:
  - node_token: 节点在知识库中的 token
  - obj_token: 文档实际 token（用于导出）
  - obj_type: "docx" | "doc" | "sheet" | "bitable" | "file" | "mindnote" | "slides"
  - node_type: "origin" | "shortcut"
  - has_child: 是否有子节点
  - title: 文档标题
  - parent_node_token: 父节点 token
  - obj_edit_time: 最后编辑时间
```

**遍历策略**: 递归 DFS，使用 channel 控制并发。has_child=true 时继续遍历子节点。

### 3.3 文档导出（核心3步）

**Step 1: 创建导出任务**

```
POST https://open.feishu.cn/open-apis/drive/v1/export_tasks
Authorization: Bearer <user_access_token>
Content-Type: application/json

{
  "token": "<obj_token>",
  "type": "docx" | "sheet" | "bitable",
  "file_extension": "docx" | "pdf" | "xlsx" | "csv"
}
```

**Step 2: 轮询导出结果**

```
GET https://open.feishu.cn/open-apis/drive/v1/export_tasks/<ticket>?token=<obj_token>
Authorization: Bearer <user_access_token>

轮询策略:
  - 创建后等待 3 秒
  - 之后每次间隔 2 秒
  - 最多轮询 60 次（超时 2 分钟）
  - status: "pending" | "processing" | "success" | "failed"
  - success 时返回 file_token
```

**Step 3: 下载导出文件**

```
GET https://open.feishu.cn/open-apis/drive/v1/export_tasks/file/<file_token>/download
Authorization: Bearer <user_access_token>

⚠️ 必须在导出任务完成后 10 分钟内下载，否则文件被删除
```

### 3.4 文档内容读取（用于 MD 转换）

```
GET https://open.feishu.cn/open-apis/docx/v1/documents/<docx_token>/raw_content
Authorization: Bearer <user_access_token>

⚠️ 飞书官方不直接支持导出 .md，需自行转换
```

## 四、导出引擎设计

### 4.1 导出流程

```
[遍历节点树]
    ↓
[构建导出队列] (只包含可导出节点，排除 folder/shortcut)
    ↓
[并发创建导出任务]  max_concurrent = N（默认 5）
    ↓
[轮询任务状态]  每个任务独立轮询
    ↓
[下载文件 & 写入临时目录]
    ↓
[如需 .md 则转换]
    ↓
[按目录结构打包 .zip]
    ↓
[清理临时文件 & 返回 zip 路径]
```

### 4.2 并发控制

```rust
// 使用 semaphore 控制并发数
let sem = Arc::new(Semaphore::new(concurrency));

// 每个导出任务获取一个许可
let _permit = sem.acquire().await?;
let ticket = create_export_task(&client, &doc).await?;

// 独立轮询（tokio::spawn）
let handle = tokio::spawn(poll_and_download(client, ticket, doc.token));

// join! 等待所有任务完成
let results = join_all(handles).await;
```

### 4.3 限流应对

| 接口 | 限制 | 策略 |
|------|------|------|
| wiki API | 100次/分钟 | 遍历时串行 + 200ms 间隔 |
| 导出任务创建 | 100次/分钟 | 并发数 ≤ 5 自动满足 |
| 导出任务查询 | 100次/分钟 | 独立轮询，指数退避 |
| 导出文件下载 | 100次/分钟 | 串行下载 |

### 4.4 Markdown 转换方案

飞书官方 API **不直接支持导出 .md**，本工具采用以下方案：

```
1. 调用飞书 API 导出 .docx
2. 使用纯 Rust docx crate 解析 .docx 文件（ZIP 包 + XML）
3. 将文档结构化块转换为 Markdown 字符串
4. 图片提取为 Base64 嵌入 Markdown
```

**实现**: `crates/doc-converter/src/lib.rs`，纯 Rust 实现，无外部依赖（如 pandoc）。

### 5.3 Token 管理

```rust
struct TokenStore {
    access_token: String,
    refresh_token: String,
    expires_at: DateTime<Utc>,
}

impl TokenStore {
    // 读取时检查是否过期，过期则自动刷新
    fn get_valid_token(&mut self) -> Result<&str> {
        if self.is_expired() {
            self.refresh().await?;
        }
        Ok(&self.access_token)
    }
}
```

## 六、错误处理

### 6.1 错误类型

```rust
enum FeishuError {
    ApiError { code: i32, msg: String },
    TokenExpired,
    PermissionDenied { node: String },
    ExportTimeout { token: String },
    FileTooLarge { size: u64 },
    RateLimited { retry_after: u64 },
    NetworkError,
    StorageError,
}
```

### 6.2 重试策略

| 错误类型 | 策略 |
|---------|------|
| `RateLimited` | 等 retry_after 秒后重试，最多 3 次 |
| `ExportTimeout` | 指数退避重试（3s, 6s, 12s） |
| `PermissionDenied` | 跳过，记录日志，继续导出其他文档 |
| `FileTooLarge` | 跳过，记录日志，继续导出其他文档 |
| `NetworkError` | 等 5 秒重试，最多 3 次 |
| `ApiError(code != 0)` | 根据错误码判断，131006 跳过，其他重试 |

## 七、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| CLI 框架 | clap (derive) | Rust 生态最成熟，derive 宏写命令 |
| HTTP 客户端 | reqwest | 异步支持好，API 友好 |
| 异步运行时 | tokio | 最成熟，生态完整 |
| JSON 处理 | serde + serde_json | Rust 标准 |
| TOML 配置 | toml | 配置存储 |
| 进度条 | indicatif | tokio 集成好 |
| 彩色输出 | console + colored | 跨平台 |
| zip 打包 | zip | 纯 Rust，无外部依赖 |
| docx→md | 纯 Rust docx crate | 纯 Rust，ZIP+XML 解析，无外部依赖 |
| Web 服务器 | tiny_http | 轻量，OAuth callback 够用 |
