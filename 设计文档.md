# 飞书文档批量导出工具 — 设计文档

**项目名称**: feishu-export
**版本**: v0.1.0
**最后更新**: 2026-04-25

---

## 一、产品概述

### 1.1 问题背景

企业用户从飞书迁移文档到本地时，面临以下痛点：
- 手动一个个导出，速度慢，易出错
- 飞书官方只支持单文档导出，无批量能力
- 目录结构无法保留，导出后难以整理
- 部分文档类型（表格、多维表格）导出后格式丢失

### 1.2 目标用户

- 企业 IT 管理员：负责文档迁移
- 项目经理：备份项目文档
- 开发者：本地编辑或接入其他系统

### 1.3 产品定位

**轻量级 CLI 工具**，一句话搞定飞书文档批量导出。不做 Web 界面，不做后台服务，专注导出这一件事。

## 二、功能需求

### 2.1 核心功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| OAuth 用户授权 | P0 | 扫码登录，用户无感 |
| 知识空间浏览 | P0 | 查看所有可访问的知识库 |
| 文档树查看 | P0 | 可视化目录结构 |
| 批量导出 | P0 | 全量导出指定知识空间 |
| 目录结构保留 | P0 | 原样重建文件夹层级 |
| 格式选择 | P1 | docx / pdf / md / xlsx / csv |
| 导出进度 | P1 | 实时显示导出进度 |
| 断点续导 | P1 | 中断后可继续，不重复下载 |
| 指定节点导出 | P2 | 仅导出特定文档或文件夹 |

### 2.2 用户操作流程

```
首次使用:
  1. 配置 App 凭证（一次性）
  2. 扫码授权（一次性）
  3. 列出知识空间
  4. 选定目标空间
  5. 开始导出

日常使用:
  1. feishu-export export <space_id>
  2. 等待导出完成
  3. 获取 .zip 文件
```

### 2.3 边界场景

| 场景 | 处理方式 |
|------|---------|
| 文档无导出权限 | 跳过，记录，跳到下一个 |
| 导出超时（>2分钟） | 标记失败，重试 3 次，仍失败跳过 |
| 图片超 1GB（docx）/ 128MB（pdf） | 跳过，记录失败原因 |
| 思维导图/幻灯片 | 不支持，提示用户手动处理 |
| 导出过程中网络中断 | 保存进度，重启后 resume |
| 磁盘空间不足 | 检测可用空间，不足则提前终止 |
| 文件名非法字符 | 自动替换（Windows 文件名禁止字符）|

### 3.2 交互模式

**非交互模式（CI/CD）**:

```bash
# 所有输出到 stdout，错误到 stderr
feishu-export export <space_id> --format md --output ./exports 2>&1
echo $?  # 0=成功，1=部分失败，2=完全失败
```

**JSON 机器可读输出（可选）**:

```bash
feishu-export export <space_id> --format md --json
# 输出结构化 JSON，便于脚本解析
```

## 四、数据结构

### 4.1 WikiNode 模型

```rust
struct WikiNode {
    space_id: String,          // 知识空间 ID
    node_token: String,         // 节点在知识库的 token
    obj_token: String,          // 文档实际 token（导出用）
    obj_type: ObjType,          // 文档类型
    node_type: NodeType,        // origin | shortcut
    title: String,              // 文档标题
    has_child: bool,            // 是否有子节点
    parent_node_token: Option<String>,
    depth: u32,                 // 树深度（用于缩进）
}

enum ObjType {
    Docx,      // 新版文档
    Doc,       // 旧版文档
    Sheet,     // 电子表格
    Bitable,   // 多维表格
    File,      // 文件
    Mindnote,  // 思维导图
    Slides,    // 幻灯片
    Folder,    // 文件夹
}
```

### 4.2 ExportTask 模型

```rust
struct ExportTask {
    node: WikiNode,
    format: ExportFormat,
    status: ExportStatus,
    progress: ExportProgress,
}

enum ExportStatus {
    Pending,       // 等待中
    Creating,      // 创建导出任务中
    Polling,       // 轮询中
    Downloading,    // 下载中
    Converting,    // 格式转换中（如 md）
    Completed,     // 完成
    Failed(String),// 失败（含原因）
}

struct ExportProgress {
    total: usize,
    completed: usize,
    skipped: usize,
    failed: usize,
}
```

## 五、非功能性需求

### 5.1 性能

| 指标 | 目标 |
|------|------|
| 单文档导出（docx） | < 10 秒（网络正常） |
| 100 个文档导出 | < 30 分钟 |
| CLI 冷启动时间 | < 1 秒 |
| 内存占用（idle） | < 20 MB |
| 内存占用（100 并发） | < 200 MB |

### 5.2 兼容性

| 平台 | 支持版本 |
|------|---------|
| macOS | 10.15+ (Catalina 及以上) |
| Linux | Ubuntu 18.04+, Debian 10+ |
| Windows | Windows 10+ |

### 5.3 安全性

- App Secret 仅本地存储，不上传
- Token 存储在用户本地配置目录，权限 0600
- 支持 `feishu-export logout` 清除所有存储数据
- 授权码一次性使用，防止 replay 攻击
- state 参数防止 CSRF

## 六、权限配置清单

在飞书开发者后台创建自建应用后，需开通以下权限：

### 6.1 必需权限

| 权限名称 | scope | 用途 |
|---------|-------|------|
| 查看知识空间列表 | `wiki:space:retrieve` | 列出可访问的知识库 |
| 查看知识空间节点列表 | `wiki:node:retrieve` | 获取文档树 |
| 查看知识空间节点信息 | `wiki:node:read` | 获取文档详情 |
| 查看新版文档 | `docx:document:readonly` | 读取文档内容 |
| 导出云文档 | `drive:export:readonly` | 创建导出任务 |
| 离线访问 | `offline_access` | 获取 refresh_token |

### 6.2 可选权限

| 权限名称 | scope | 用途 |
|---------|-------|------|
| 查看、评论、编辑和管理电子表格 | `sheets:spreadsheet` | 导出 Excel |
| 查看、评论、编辑和管理多维表格 | `bitable:app` | 导出多维表格 |
| 查看云空间文件 | `drive:drive:readonly` | 下载文件类型文档 |

### 6.3 知识库权限配置

仅开通 API 权限还不够，还需将应用添加为知识库管理员：

1. 在飞书桌面端打开目标知识库
2. 进入「知识空间设置」→「成员管理」→「添加管理员」
3. 搜索并添加机器人（自建应用名称）
4. 机器人需先添加「机器人」应用能力
